hexo远端部署

一.注册github账号

登入github官网，注册账号。

完成之后在本地配置个人信息

（将your_name和your_email换成自己的信息）

二.创建github仓库

打开github的个人主页，点击New repository 创建仓库，Repository name 的格式为 username.github.io，其中，username 是自己的 GitHub 名称

另外，建议勾选 Initialize this repository with a README 选项，生成 README.md 文件

最后点击 Create repository，完成创建仓库

三.配置

进入站点根文件夹，打开 git bash 输入 npm install hexo-deployer-git --save 命令安装插件

打开根文件夹下的 _config.yml 配置文件，进行如下配置

四.配置SSH连接

ssh连接相对https连接具有更稳定更安全等特点，且SSH每次部署的时候不需要输入用户名与密码。

进入站点根文件夹，打开 git bash，输入 ssh-keygen -t rsa -C "your_email" 命令并一路回车获得密钥

完成后，根据提示信息中的文件地址 /c/Users/user/.ssh/id_rsa.pub 找到 id_rsa.pub 文件，复制文件里面的内容

登录 Github，打开 Settings

选择 SSH and GPG keys，选择 New SSh Key

将复制的内容添加到 Key 中， Title 可以自己设定，最后点击 Add SSH Key 即可

最后，回到站点根文件夹，打开 git bash，输入 ssh -T git@github.com 命令验证是否添加成功

hexo系列1

发表于 2025-06-19 分类于 hexo

安装之路

一.hexo简介

Hexo 是一个快速、简洁且高效的 博客框架。它使用 Markdown解析文章，在数秒内，即可使用靓丽的主题生成静态网页。

二.前置程序安装

1.git安装

Git 是一个开源的分布式版本控制系统，用于跟踪文件和目录的变更历史，广泛应用于软件开发领域。

下载地址：git官网下载 所有选项默认安装即可

当右键鼠标时，出现图中的 GIt Bash Here时代表安装成功。

2.安装Node.js

Node.js 是一个开源的、跨平台的 JavaScript 运行时环境，它允许开发者在服务器端运行 JavaScript 代码。npm是 Node.js 的官方包管理工具，同时也是全球最大的软件包注册表之一。它为 Node.js 提供了丰富的第三方模块支持，极大地简化了 JavaScript 项目的依赖管理和开发流程。

下载地址：Node.js官网下载 同样一直默认安装即可

鼠标右键桌面，点击 Git Bash Here 进入Bash界面，输入node -v 和npm -v命令

出现版本号代表安装成功

3.安装Hexo

前面说过npm是 Node.js 的官方包管理工具，利用npm来安装hexo框架。

npm install moduleName 命令用于安装模块，其中 -g 参数代表在全局进行安装。

在bash窗口用这个命令进行hexo的安装，利用hexo -v 命令查看hexo版本号，出现版本号代表安装成功。

三.hexo本地建站

1.本地建站

选择一个合适的位置创建一个文件夹，该文件夹将作为 Hexo 的根目录，用来储存本地文件，

进入文件夹，打开Bash窗口，输入 hexo init 和 npm install 命令

hexo init 命令可以将该文件夹初始化为Hexo的根目录

npm install 命令可以自动安装依赖列表中列出的所有模块

2.测试

在Hexo根目录下进入Bash窗口。

输入 hexo server 命令，启动本地站点

出现如图，表示启动成功，再使用浏览器输入http://localhost:4000/ 即可访问本地站点
