配置文件详解

1.site

网站的个性化描述

setting 描述
title 网站标题
subtitle 网站副标题
description 网站描述
keywords 网站关键词
author 网站作者
language 网站使用的语言，默认是en ，中文网站填zh-Hans
timezone 网站使用的时区，默认为 计算机的预设置，可以不填

2.url

关于博客文章 URL 的设置，一般不用进行更改

setting 描述
url 网站的网站
permalink 文章的链接格式 ，默认为 :year/:month/:day/:title/
pretty_urls trailing_index：设置为 true，表示在永久链接的末尾保留 index.html。如果设置为 false，则会移除 index.html，使 URL 更简洁。 trailing_html：设置为 true，表示在永久链接的末尾保留 .html。如果设置为 false，则会移除 .html，使 URL 更简洁。

3.Directory

关于文件夹的设置，也是一般不用进行更改

setting 描述
source_dir 资源文件夹 ，存放用户的资源文件，默认为 source
public_dir 公用文件夹 ，存放生成的静态文件，默认为 public
tag_dir 标签目录 ，默认为 tags
archive_dir 档案目录 ，默认为 archives
category_dir 分类目录 ，默认为 categories
code_dir 代码目录 ，默认为 downloads/code
i18n_dir i18n目录 ，默认为 :lang
skip_render 储存站长验证文件，跳过指定文件的渲染

4.Writing

这里是比较常用的写作设置，可以根据自己的写作习惯随时进行调整

setting 描述
new_post_name 新文章的文件名格式，默认为 :title.md
default_layout 预设的布局模板，默认为 post
titlecase 标题是否使用首字母大写 ，默认为 false
external_link enable：true ， 是否在新标签页中打开外部链接。设置为 true 可以提高用户体验，避免用户离开当前页面。 field：site， 应用范围。site 表示应用于整个站点。 exclude：''， 排除的域名。如果某些外部链接不需要在新标签页中打开，可以在这里列出这些域名。
filename_case 将文件名转换为 1 小写 或 2 大写，默认为 0
render_drafts 是否显示渲染草稿，默认为 false
post_asset_folder 是否启用文章资源文件夹。设置为 true 表示每篇文章都会有一个对应的资源文件夹，方便管理文章中的图片、文件等资源。
relative_link 是否使用相对链接。设置为 false 表示不使用相对链接，链接将基于站点的根目录。
future 是否显示未来日期的文章。设置为 true 表示允许显示发布日期在未来的文章。
syntax_highlighter 代码高亮工具。highlight.js 是一个流行的代码高亮库，支持多种编程语言
highlight line_number：true，是否显示代码行号。设置为 true 可以在代码块中显示行号，便于阅读和调试。 auto_detect：false ，是否自动检测代码语言。设置为 false 表示不自动检测，需要手动指定代码语言。 tab_replace：'' ，替换制表符的字符。默认为空，表示不进行替换。 wrap：true ，是否对代码块进行包裹。设置为 true 可以使代码块在页面中更好地显示。 hljs：false ，是否使用 highlight.js 的高级功能。设置为 false 表示不使用。
prismjs preprocess：true ，是否预处理代码块。设置为 true 可以提高代码高亮的性能。 line_number：true ，：是否显示代码行号。设置为 true 可以在代码块中显示行号。 tab_replace：'' ，：替换制表符的字符。默认为空，表示不进行替换。

5.Home page setting

首页设置，可以自己决定每页显示的文章数量和显示文章的顺序，index_generator——主页设置

setting 描述
path 默认值：‘’。首页的根路径。默认值为空字符串，表示首页路径为网站的根目录。如果您希望将首页放在一个特定的子目录中，可以在这里指定路径，例如 home，则首页的路径将是 https://yourdomain.com/home。
per_page 默认值：10。每页显示的文章数量。设置为 10 表示每页显示 10 篇文章。如果您希望禁用分页，可以将此值设置为 0。
order_by 默认值：-date 。文章的排序方式。默认值为 -date，表示按日期降序排列（最新的文章在前）。如果您希望按日期升序排列（最旧的文章在前），可以将其设置为 date。您还可以使用其他字段进行排序，例如 updated 或 title。

6.Category & Tag

这里是关于分类和标签的配置

setting 描述
default_category 预设分类，默认为 uncategorized
category_map 分类别名
tag_map 标签别名

7.meta_generator

meta_generator 是一个用于控制是否在生成的 HTML 文件中包含 <meta> 元素的设置。

seeting 描述
meta_generator 默认值：true ，当设置为 true 时，Hexo 会在生成的 HTML 文件的 <head> 部分添加一个 <meta name="generator" content="Hexo"> 标签。这个标签表明网站是使用 Hexo 生成的。如果您希望隐藏这个信息，可以将其设置为 false。

8.Date / Time format

时间和日期的显示格式，一般没特殊要求的也不需要改

setting 描述
date_format 日期格式，默认为 YYYY-MM-DD
time_format 时间格式，默认为 HH:mm:ss

9.Pagination

这里是分页设置，可以自己决定单个页面上显示的文章数量和分页目录

setting 描述
per_page 单个页面上显示的文章数量，默认为 10 ，用 0 表示禁用分页
pagination_dir 分页目录，默认为 page

10.Include / Exclude file(s)

include、exclude 和 ignore 是用于控制哪些文件和文件夹应该被包含或排除在生成的静态文件之外的设置。

setting 描述
include 指定哪些文件或文件夹应该被包含在生成的静态文件中。这通常用于包含一些默认情况下会被 Hexo 忽略的文件，例如隐藏文件（以 . 开头的文件）或特定的文件类型。默认值：空数组 []
exclude 指定哪些文件或文件夹应该被排除在生成的静态文件之外。这通常用于排除一些不需要生成的文件，例如临时文件、备份文件或特定的文件类型。默认值：空数组 []
ignore 指定哪些文件或文件夹应该被完全忽略，既不会被渲染也不会被复制到 public/ 文件夹中。这通常用于排除一些不需要处理的文件，例如临时文件、配置文件等。默认值：空数组 []

11Extensions

这里可以设置主题类型和插件

setting 描述
theme 博客使用的主题，默认为 landscape

12.Deployment

这里是关于网站部署的配置，常用的有部署类型和部署地址

setting 描述
type 网站部署类型
repo 网站部署地址
branch 目标分支的名称
